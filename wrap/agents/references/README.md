# References

This directory contains reusable AI reference material for the WRAP system.

References are not step-by-step workflows and are not copy-paste code templates. They capture durable project techniques, conventions, and operational notes that multiple workflows can point to.

## Usage

Use references when a task needs background technique guidance in addition to a workflow.

Examples:

- Browser QA tactics that apply across pull requests.
- Project-specific external service interaction notes.
- Cross-workflow conventions that do not belong to one implementation plan.

Keep task procedures in [`../workflows/`](../workflows/), implementation plans in [`../plans/`](../plans/), and code examples in [`../templates/`](../templates/).
