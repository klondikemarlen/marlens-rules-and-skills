# Release Notes

Follow the WRAP project workflow in `agents/workflows/release-notes-workflow.md` when that file exists in the current repository.

Read that workflow before drafting or editing WRAP release notes. Treat it as the source of truth for style, required fields, source priority, deployment-context wording, and no-overclaiming rules.

If the workflow file is not present, ask for the project-specific release-notes workflow or use the user's provided release-note style instructions.
